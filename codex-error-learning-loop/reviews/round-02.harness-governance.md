# 周度 Harness 学习闭环治理评审

- Skill：`codex-error-learning-loop`、`workflow-pattern-retrospector`、`codex-task-continuity`
- 结论：更新已有 Skill；不创建重复 Skill。
- 理由：问题分别属于纠错事实记录、跨周模式复盘和摘要投递，已有能力边界清楚，但缺少结构化状态和闭环连接。
- 需要修改：三个 Skill、全局 Hook、周一 Automation、Obsidian 索引和变更日志。
- 不需要修改：不把完整流程复制进全局 `AGENTS.md`；现有入口规则已足够发现相关 Skill。
- 验证建议：观察误报/去重、跨线程门槛、上一自然周边界、候选生命周期、真实周总结投递。

## 对抗式审查

- **误报风险**：Hook 只记录明确纠正，且观察状态不触发改动。
- **过拟合风险**：同线程重复不增加独立证据；普通候选至少跨两个线程。
- **自修改风险**：Hook 永不改规则；周度 Agent 只有经过治理和回归场景才可启动小范围试验。
- **可见性风险**：周总结必须投递到摘要归档线程，内部报告路径不作为用户结论。
- **陈旧候选风险**：候选有状态、最后出现时间和下一检查时间，可被拒绝或验证关闭。

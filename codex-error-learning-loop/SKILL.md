# Codex 错误学习闭环

把用户纠错和已确认失败转成可复用的防复发规则。默认事件驱动，不自动审判每轮对话，不自动修改全局规则。

## 触发判断

使用本 skill：

- 用户明确指出“你理解错了”“不是这个项目”“你发散了”“应该先问我”“你反复改还是不对”“你说验证了但没有验证”等。
- 复盘线程、Hook 摘要、任务账本或调研报告时，需要提取错误模式。
- 搜索或调研漏掉用户指出的明显候选，需要修正检索策略。
- 准备把重复错误沉淀为 Skill、项目规则、全局规则、Hook 或 Obsidian 记录。

不要使用本 skill：

- 只是普通代码 review、语法修复或用户改主意。
- 错误尚未确认，只是模型自我猜测。
- 用户只想继续完成当前任务，不希望做复盘；这时先修正当前任务，最后用一句话记录可选教训。

## 工作流程

1. 先修正当前任务

   如果错误影响正在执行的任务，先停下错误路径，确认新的目标、范围、项目目录和验收标准。不要一边继续实现一边扩大复盘范围。

   当纠错改变了需求、范围、接口、数据、项目归属或验收标准时，回到相关 spec、plan 或执行契约更新；不要只做局部补丁。

2. 写错误学习卡片

   用精简卡片记录事实，不复制长 transcript，不记录 secrets。

   | 字段 | 说明 |
   |---|---|
   | 触发信号 | 用户原话的短摘或事实描述 |
   | 原始目标 | 用户真正要达成的目标 |
   | 错误假设 | Codex 当时错误理解了什么 |
   | 错误类型 | 从下方分类中选择 1-3 个 |
   | 影响范围 | 当前回答、当前文件、当前项目、跨项目、长期规则 |
   | 应提前确认 | 如果重来，一开始该问或该检查什么 |
   | 防复发规则 | 下次遇到同类场景应执行的具体动作 |
   | 证据 | 文件、线程、命令、用户纠正或验证结果 |
   | 处理状态 | 候选、已确认、已修正、已晋升、无需晋升 |

3. 分类错误类型

   | 类型 | 判断标准 |
   |---|---|
   | 需求误读 | 把用户目标、例子或约束理解错 |
   | 未澄清就执行 | 应先问或先查，却直接实现或下结论 |
   | 范围漂移 | 做了用户没要求的扩展、重构或文档化 |
   | 跨项目误改 | 在错误目录、错误仓库或错误项目空间操作 |
   | 上下文污染 | 被旧线程、旧目标或无关规则带偏 |
   | 反复返工 | 多轮修补暴露出前置判断缺失 |
   | 验证声明不实 | 没有跑验证、验证失败或证据不足却声称完成 |
   | 流程过重或过轻 | 小任务套重流程，或高风险任务缺少契约和验证 |
   | 搜索漏召回 | 用户给的明显线索或同义关键词没有进入搜索策略 |

4. 判断晋升位置

   | 情况 | 动作 |
   |---|---|
   | 一次性、低风险 | 只在当前回复中给卡片或简短教训，不改文件 |
   | 已确认但还不稳定 | 写入任务/项目复盘或 Obsidian 候选记录 |
   | 同类错误重复 2 次以上 | 更新相关 Skill 的失败模式或检查点 |
   | 只影响当前仓库 | 更新项目 `AGENTS.md` 或项目内 Skill |
   | 跨项目高频、默认入口或安全边界 | 先用 `skill-governance-review`，再考虑全局 `AGENTS.md` |
   | 需要自动捕获 | 先做 Hook 方案和停止条件；Hook 只产候选，不自动晋升 |

   涉及 Skill 新增、更新、安装、归档、改名或删除时，必须先使用 `skill-governance-review`。涉及 Obsidian 写入时，先读取 Vault 根目录 `AGENTS.md`，并遵守其 PARA 写入规则。

5. 选择后续工具

   | 场景 | 后续入口 |
   |---|---|
   | 搜索漏召回或用户例子被误判 | 使用 `requirement-first-research` 重新拆需求和查询矩阵 |
   | 参考项目已筛出，需要落地路线图 | 使用 `reference-project-study-roadmap` |
   | 需要更新或创建 Skill | 使用 `skill-governance-review` 和 `skill-creator` |
   | 需要周期性发现重复流程 | 使用 `workflow-pattern-retrospector` |
   | 需要写长期记忆 | 使用 `obsidian-memory-workflow` |

## 输出格式

默认输出一段简短结论，加一张卡片：

```markdown
结论：这是已确认/候选错误，建议先修正当前任务，暂不/需要晋升。

| 字段 | 内容 |
|---|---|
| 错误类型 | ... |
| 错误假设 | ... |
| 应提前确认 | ... |
| 防复发规则 | ... |
| 晋升建议 | 候选 / 更新 Skill / 更新项目规则 / 治理后更新全局规则 |
```

如果用户只需要继续任务，把卡片压缩成 3-5 行，不要打断主流程。

## 安全边界

- 不记录 secrets、tokens、私钥、Cookie、账号密码、`.env` 值。
- 不把用户纠错写成对用户的归因；只记录 Codex 可改变的行为。
- 不把一次错误直接升级成全局规则。
- 不自动安装外部 Skill，不复制许可证不清的 Skill 正文。
- 不让 Hook 自动修改 Obsidian、Skill 或 `AGENTS.md`；Hook 最多生成候选事实。

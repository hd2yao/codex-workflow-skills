# Codex 每日连续性增强 Spec

## 目标

每日摘要必须回答三件事：前一天实际做了什么、交付后还需要什么、周期任务是否按计划运行。仓库区只展示已执行动作和真正阻塞，不把原始 Git 事实全部转交给用户判断。

## 验收标准

- **AC-001**：每日自动化读取前一自然日实际活跃的 Codex 任务，并记录为结构化活动；不再只依赖 task/work ledger。
- **AC-002**：活动状态至少区分已完成、已交付待试用、调研完成待实施、进行中、等待用户和阻塞，并展示具体下一步。
- **AC-003**：项目可用 `.codex/continuity.json` 声明周期任务；审计器结合计划时间、调度器、退出码、结构化状态和日志新鲜度判断 `success`、`overdue`、`failed` 或 `unknown`。
- **AC-004**：`ya-fundmind` 的 daily/weekly 被日报识别，daily 显示最近运行日期、业务状态和数据质量，weekly 显示最近证据和下一次计划时间。
- **AC-005**：仓库扫描分别输出相对默认分支和 upstream 的 ahead/behind，不再混用比较基准。
- **AC-006**：高置信已合并分支可自动清理本地 worktree、本地分支和远端分支；有唯一提交的干净分支先 push 保存；分叉、脏改动和冲突停止自动合并。
- **AC-007**：仓库写操作预算从 3 起步，连续 7 次合格运行可按 `3 -> 5 -> 8 -> 10` 提升，风险事件触发回退；只读扫描不受该预算限制。
- **AC-008**：所有自动推断保留证据来源和时间；缺少新鲜证据时不得报告“正常运行”。
- **AC-009**：线程索引失败时，从操作日志恢复昨日项目活跃记录和已核实工作流变更，并从关联上下文卡片提取最近项目进展；状态仍保持进行中，不得把活跃或进展事件误写成完成成果。
- **AC-010**：非 Git 正式项目内部的源码和文档自动归属，不进入待确认池；识别为正式内容后不得继续显示删除类操作。
- **AC-011**：日报不重复展开旧 work ledger 条目；昨日 Skill、Hook、Automation、Plugin 和项目活动优先展示，历史成果只保留索引入口。
- **AC-012**：目标因时间、调度器、CI、外部审批或其他可观察条件等待时，记录结构化续作监控，至少包含目标、等待条件、恢复动作、恢复方式、监控 Automation、下次检查和可并行工作。
- **AC-013**：日报单列“等待条件与续作监控”，明确哪些目标自动监控、哪些条件已满足、哪些监控缺失或逾期、用户是否需要操作；不能只展示关联周期任务成功。
- **AC-014**：单个 gate 等待时，如果存在安全且隔离的并行工作，目标保持执行中并切换并行轨道；只有所有轨道均不可推进时才使用 `blocked`。
- **AC-015**：自动续作必须绑定原线程和 ACTIVE Automation；Automation 缺失、停用、投递线程不匹配或超过下次检查时间未回写时，日报升级为需要处理，不能静默等待。
- **AC-016**：昨日工作按“项目名 / 状态 / 昨日结果 / 下一步”展示；项目名是第一识别信息，线程标题只作辅助，单项不得展开上下文压缩原文。
- **AC-017**：用户卡片不显示证据路径、内部仓库报告入口、稳定 RC 编号或“读取证据后再判断”等内部操作；证据仍保留在 JSON 和内部报告供自动化复核。
- **AC-018**：仓库发现必须给出明确阶段和原因，至少区分未提交改动、已提交未推送、已推送未合并、分叉、PR 未完成、已合并待清理和自动动作失败；禁止使用“证据不足”作为用户可见结论。
- **AC-019**：支持持久忽略非本人、只读镜像或明确不再管理的仓库；忽略项不 fetch、不进入发现、不产生扫描警告，也不出现在日报。
- **AC-020**：最近活动超过 15 天且没有近期任务继续证据的仓库默认进入自动收尾；近期仍在开发时暂不合并，但日报必须写明对应项目、分支、当前阶段和继续条件。
- **AC-021**：仓库候选优先按精确 cwd / worktree / 仓库路径关联 Codex 任务；可关联时由对应任务分析或完成收尾，无法关联时由每日自动化直接处理，不能把关联工作转交用户。
- **AC-022**：每日自动化先完成仓库处置、记录结构化结果、复扫，再展示最终摘要；初始扫描结果不先展示。最终卡片只分为今日已处理、近期开发暂缓和确切失败。
- **AC-023**：三仓写预算按仓库而不是 finding 计数，并优先一次消除同仓库多个历史项；只读分析、忽略和任务交接不占写预算。

## 安全边界

- 周期任务审计只读取声明、调度器状态、日志元数据和白名单 JSON 字段，不读取 `.env` 或业务文件正文。
- 线程活动只存标题、状态、摘要、下一步、项目路径和短证据，不保存完整 transcript。
- 删除分支前必须 fresh fetch/prune、工作区干净、无开放 PR，并有提交可达、内容等价或已合并 PR 证据。
- 动态预算不得超过 10，不因没有处理项而升级。
- 续作监控只保存短目标、条件、动作、项目/线程标识和 Automation 元数据，不保存完整 transcript 或 Automation prompt。
- 自动收尾不得 force push、猜测冲突、提交 secrets 或绕过仓库保护；超过 15 天只改变默认处置优先级，不取消测试、diff review、远端身份和可回滚门槛。
- 用户可见摘要省略证据不等于删除证据；完整事实继续保留在内部 JSON、操作日志和仓库报告中。
